Проект учебный и предназначен в первую очередь для практики и отработки базовых frontend-паттернов!
Сделан для практики архитектуры фронтенд-приложения, работы с DOM и ООП-подхода без фреймворков.

## О проекте
- это тренировочный мини-конструктор, где контент страницы формируется из блоков.  
Часть блоков задается в стартовой конфигурации, а новые можно добавлять через боковую панель.

Проект используется как полигон для практики:
- декомпозиции кода на модули;
- работы с классами и наследованием;
- рендера интерфейса через динамическую генерацию HTML;
- передачи стилей через конфигурацию.

## Основные фичи

- Рендер контента из массива блоков (`Title`, `Text`, `Columns`, `Image`).
- Базовый класс `Block` и специализированные классы с собственным `toHtml()`.
- Динамическое добавление новых блоков через форму в `Sidebar`.
- Мгновенное обновление страницы после добавления блока (callback + повторный render).
- Настройка инлайн-стилей для блоков через объект или строку.
- Простая, наглядная структура проекта для обучения.

## Какие методы и подходы применены

- **ООП**: инкапсуляция логики в классах `Site`, `Sidebar`, `Block` и его наследниках.
- **Наследование и полиморфизм**: общий интерфейс `toHtml()` в дочерних блоках.
- **Модульная архитектура (ES-модули)**: разбиение на `classes`, `utils`, `modal`.
- **Работа с DOM API**: `querySelector`, `insertAdjacentHTML`, обработчики событий.
- **Event-driven подход**: обработка `submit` и реактивное обновление состояния.
- **Конфигурируемость**: блоки создаются из данных и настроек, а не захардкожены в шаблон.
